# Gemini Workspace Instructions

1. **Technology Stack**: Use Go for the server application.
2. **Architecture**: Use a clean and organized project structure following Domain-Driven Design (DDD) or Clean Architecture patterns. Define interfaces for infrastructure layers (e.g., cursor control interfaces, WebSocket/HTTP server abstraction) to decouple them from domain logic.
3. **Strict Adherence to Scope**: Adhere strictly to the explicit scope of every user request. Do not perform any actions, create any files, or implement any logic beyond what is specifically and directly instructed.
4. **No Over-Initiative**: Do not make assumptions about next steps. Wait for explicit instructions before creating files, running tests, or initiating new development phases.
